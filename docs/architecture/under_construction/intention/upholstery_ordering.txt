and a ordered amount.

oder fields to the upholstery order can come in the request, like:
supplier_id, price_minor, currency, order_at, expected_recieve_at.
the default state on creation is of "ordered" but the request can override it for ( "draft", "pending", "approved", "ordered")

when creating the instance it will use the passed amount as a resouce to mark the upholstery requirements that have state needs_ordering to state ordered.
this will be done in order of priority, first is the list of ids passed,
then it is the upholstery requirements where the task.ready_by_at is closest to the current date ( in case of a ready_by_at tie it solves for the oldest record ).
then it is the oldest upholstery requirement creation time .
