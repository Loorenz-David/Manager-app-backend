the cases model will now have a column called reference_number and a column called scalar_id , both with indexes.
the scalar_id will be an scalar_id which it is increased on each case instance creation, i believe postgres porvides us with a way to build this reliably.
task model uses it.

on creation of a case service  create_case  we will make the service to check the latest scalar_id case ( the bigest as that is the latest  ) 
and we will build a reference_number with that scalar_id, the reference_number has the format of "{index}-000{scalar_id}" where the 000 get ocupy by the size of the scalar_id if scalar_id is 10 then "00{scalar_id}" , 
i scalar_id grows beyond the four digits we add an extra digit. 
the "index" is taken from the case_link entity_client_id index if no index on the entity_client_id then simply a N as index.

we will add this scalar_id and reference_number on the serialization of the cases.
