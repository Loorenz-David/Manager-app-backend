- Added `ItemCategoryIssueType` as the new issue-type-to-item-category linkage table and removed the legacy severity/category-config model path.
- Replaced standalone item-issue create/delete commands with batch create and batch delete commands, plus a filterable `get_item_issues` query.
- Updated embedded issue payloads in item creation, task creation, serializers, routers, and task analytics to use the new snapshot shape.
- Added issue-type management commands for create, update, and batch delete, including working-section and item-category link synchronization.
- Replaced bootstrap category-config seeding with issue-type link seeding and added reset cleanup for `item_category_issue_types`.
- Generated and validated Alembic migration `99accdeba8b9_issue_system_rework.py`, including the `IssueSourceEnum.MANUAL` enum addition and the schema rollback path.
