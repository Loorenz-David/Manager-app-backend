
the upholstery table will now have a column called page_link which is a string ( can be nullable ).
the current router route_create_upholstery will now accept the key for page_link, the supplier_name name key and  all the other fields for creating a supplier ( all optional ).
this  will be used by the service create_upholstery  for adding the page_link to the upholstery instance and for creating an upholstery suplier instance on the same process if supplier provided.


the serializers use to serialize upholstery ( serialize_upholstery ) will now have the key page_link and supplier_name .
 you should trace back the services that use those serializers as we will now need to join the upholstery supplier to the query.

after the db modifications you can make the run the migration also.
